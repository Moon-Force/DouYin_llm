# Status Strip Layout Redesign

## Goal

重构顶部 [StatusStrip.vue](H:\DouYin_llm\frontend\src\components\StatusStrip.vue) 头部布局，减少当前横向平铺造成的视觉杂乱，建立清晰的操作区与状态区层级，同时保留现有功能与中英切换能力。

## Scope

本次只改顶部头部，不改：

- [App.vue](H:\DouYin_llm\frontend\src\App.vue) 以下主工作区布局
- 提词器、事件流、观众工作台的主体排布
- 后端接口与数据结构

## Current Problems

- 头部信息目前横向散排，房间控制、连接状态、模型状态、统计数据彼此割裂
- 右上角工具按钮像后贴上的控件，没有并入整体结构
- “未连接” 目前只是文字，缺乏状态层级与快速识别能力
- 输入框、模型信息、统计数字之间缺少清晰的功能分区

## Chosen Layout

采用“左主控 + 右状态”的双栏头部结构：

- 左侧主控制区：
  - 房间标题
  - 当前房间号或未选择状态
  - 房间输入框
  - 切换房间按钮
  - 连接状态标签
- 右侧状态功能区：
  - 评论区卡片
  - 模型区卡片
  - 功能区卡片

桌面端使用左右分栏，移动端改为上下堆叠。

## Card Structure

### Connection Card

左侧连接卡片为主视觉焦点，承担唯一主操作职责。

内容：

- `房间`
- 当前房间号 / `未选择房间`
- 输入框
- `切换房间`
- 状态标签

状态标签不再使用纯文字，而改为“图标/圆点 + 文案”的轻标签样式。

### Comment Card

右侧评论卡片只展示评论总数与简短标签，不再混入连接或模型信息。

### Model Card

右侧模型卡片展示：

- 模型名
- 当前结果状态
- 次级说明（错误信息或模式）
- `设置` 按钮

### Tool Card

右侧工具卡片收纳语言切换与主题切换按钮，使它们成为头部工具区的一部分，而不是浮在整体布局之外。

## Connection Status Treatment

保留“未连接”语义，但增加视觉层级。

状态展示规则：

- `idle` / `reconnecting`：红色倾向标签
- `connecting` / `switching`：黄色倾向标签
- `live`：绿色倾向标签

标签形态：

- 左侧状态点或小图标
- 右侧状态文案
- 背景使用轻透明色块
- 圆角胶囊，不与主按钮混淆

## Visual Direction

整体方向是“直播工作台头部”，而不是信息清单。

要求：

- 每个功能块使用轻量级卡片分区
- 深色模式下采用微亮底色、细边框、大圆角
- 左侧连接卡片视觉权重大于右侧三卡片
- 右侧卡片风格统一，减少漂浮感和空白断裂
- 工具按钮并入卡片体系，而不是悬浮在头部最外层

## Interaction Constraints

- 不新增新的业务交互
- 不修改现有事件、接口、store 状态结构
- 保持现有中英切换行为
- 保持现有主题切换行为

## Testing Strategy

至少覆盖：

- 连接状态文案映射仍然正确
- 状态标签在不同连接状态下能渲染对应文案
- 前端构建通过

本次优先使用现有 node-based smoke test 加上 `npm run build` 进行验证。

## Scope Guard

本次要做：

- 顶部头部双栏重排
- 连接区 / 评论区 / 模型区 / 工具区卡片化
- “未连接” 升级为状态标签样式

本次不做：

- 主内容区重排
- 新增顶部导航菜单
- 新增房间历史、快捷入口、更多操作菜单
